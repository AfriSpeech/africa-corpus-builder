# Africa Machine Translation Parallel Corpus Builder

A toolkit for building parallel text datasets from African Bible translations on
[YouVersion](https://www.bible.com). It pairs each local-language Bible verse
with its English equivalent and saves the results as clean CSVs ready for
machine-translation training and NLP research.

Unlike the per-language [`ghana-mt-builder`](https://github.com/ghananlpcommunity/ghana-mt-builder),
this edition is **fully automated**: you give it a CSV of YouVersion versions
and it scrapes **every row, one language at a time** — you never type a version
code.

---

## What it does

YouVersion hosts hundreds of Bible translations across the African continent.
Each translation is aligned verse-by-verse against a common reference, which
makes Bible text one of the best naturally-occurring sources of parallel
sentences for low-resource African languages.

This project reads a CSV of versions, scrapes the verse pairs for each one,
cleans the text, and saves them as structured CSV files — a collection of
sentence-level `(local language, English)` pairs ready to train or fine-tune
machine-translation models.

It uses bible.com's internal chapter JSON API (one request per chapter, ~30×
fewer than per-verse scraping). The English side is batch-cached the first time
a chapter is fetched and reused for every subsequent language, so English costs
near-zero extra requests across the whole run. **No Chrome / Selenium required.**

## Quick start

### Requirements

- Python 3.10 or later

> All Python dependencies are installed automatically on first run. You do not
> need to run `pip install` yourself.

### Clone and run

```bash
git clone https://github.com/michsethowusu/africa-mt-builder.git
cd africa-mt-builder

# Scrape every version listed in the default CSV:
python youversion_parallel_text_builder.py

# Or point it at your own CSV:
python youversion_parallel_text_builder.py path/to/my_versions.csv
```

The script will:
1. Install any required packages in the background.
2. Load every viable row from the CSV.
3. Scrape each language/version **automatically, one after another** — no
   prompts. For each version it probes the Old and New Testaments, then fetches
   only the testaments that have content.

### Input CSV format

| Column | Required | Notes |
|---|---|---|
| `version_id` | ✅ | Numeric YouVersion version id. Non-numeric rows are skipped. |
| `lang_code` | ✅ | ISO-ish language code, used in output filenames. |
| `lang_name` | ✅ | Human-readable name, used in output filenames. |
| `viable` | optional | Row is skipped only if this is literally `false`. |
| `abbr` | optional | Version abbreviation (not required by the API). |

Any other columns (e.g. `country`, `has_text`, `version_title`) are ignored, so
you can keep them for your own bookkeeping.

The included [`youversion_africa_versions.csv`](youversion_africa_versions.csv)
is the **full list of African-language Bible versions on YouVersion** — **1,090
text versions across 792 languages**. It was generated by cross-referencing
YouVersion's version catalogue (`nodejs.bible.com/api/bible/configuration/3.1`)
against [Glottolog](https://glottolog.org)'s `Africa` macroarea (ISO 639-3).
Columns: `version_id, lang_code, lang_name, country, abbr, has_text, version_title`
(`country` is the Glottolog ISO-3166 list for that language).

> Running it against the whole CSV is a large, multi-day scrape. To target a
> subset, just copy the rows you want into a smaller CSV and pass it as the
> argument.

### Output

```
african_bible_parallel_text_datasets/
    progress.json              # resume state (git-ignored)
    testament_status.json      # cached OT/NT probe results (git-ignored)
    english_cache.csv          # shared English verses, reused across languages
    {Lang_Name}_{code}_v{version_id}.csv
```

Each language CSV has the columns: `verse_key, version_id, eng, local`.

### Resuming an interrupted run

Progress is tracked in `african_bible_parallel_text_datasets/progress.json`. If
a run is interrupted, just run the same command again — already-completed
chapters are skipped automatically, across every language in the CSV.

> `progress.json` and `testament_status.json` are git-ignored and will not be
> committed.

## Configuration

Tunables live near the top of `youversion_parallel_text_builder.py`:

| Setting | Default | Meaning |
|---|---|---|
| `ENGLISH_VERSION_NUM` | `37` (CEB) | English reference version paired against. |
| `NUM_WORKERS` | `16` | Concurrent chapter fetchers. |
| `REQUEST_DELAY` | `2` | Seconds between requests per worker (be polite). |
| `OUTPUT_ROOT` | `./african_bible_parallel_text_datasets` | Where datasets are written. |
